## Feature Description

The Loan Calculation API is the backend core of the Loan Simulator application. It provides a RESTful API endpoint that accepts loan parameters (amount, term, interest rate), performs simple interest calculations, and returns the monthly payment amount along with a complete amortization schedule showing the breakdown of each installment.

This API serves as the single source of truth for all loan calculations and must be developed first following the API-First development approach defined in the constitution.

### Context
- **Component**: Backend (Python + FastAPI)
- **Development Approach**: Test-Driven Development (TDD)
- **Integration Point**: Frontend consumes this API via Fetch API
- **Calculation Method**: Simple Interest (not compound interest)

## Functional Requirements

### FR1: API Endpoint Definition

**Endpoint**: `POST /api/calculate-loan`

**Request Schema**:
```json
{
  "amount": number,    // Loan principal amount (positive decimal)
  "term": number,      // Loan term in months (positive integer)
  "rate": number       // Annual interest rate as percentage (0-100)
}
```

**Success Response (HTTP 200)**:
```json
{
  "monthlyPayment": number,      // Fixed monthly payment amount
  "totalPayment": number,        // Total amount paid over loan term
  "totalInterest": number,       // Total interest paid
  "installments": [
    {
      "paymentNumber": number,   // Sequential number (1 to term)
      "paymentAmount": number,   // Monthly payment (consistent)
      "principal": number,       // Principal portion of payment
      "interest": number,        // Interest portion of payment
      "remainingBalance": number // Outstanding balance after payment
    }
  ]
}
```

**Error Response (HTTP 400)**:
```json
{
  "error": "Validation Error",
  "details": {
    "amount": "Must be greater than 0",
    "term": "Must be a positive integer",
    "rate": "Must be between 0 and 100"
  }
}
```

### FR2: Input Validation Rules

| Parameter | Validation Rule | Error Message |
|-----------|----------------|---------------|
| `amount` | Required, numeric, > 0 | "Loan amount must be greater than 0" |
| `term` | Required, integer, > 0 | "Loan term must be a positive integer (months)" |
| `rate` | Required, numeric, ≥ 0, ≤ 100 | "Interest rate must be between 0 and 100" |

**Additional Validation**:
- All parameters must be present in request body
- Data types must match schema (reject non-numeric values)
- No null or undefined values accepted

### FR3: Calculation Logic (Simple Interest)

**Monthly Payment Calculation**:
```python
# Note: This is simple interest, NOT amortizing loan formula
total_interest = principal × (rate / 100) × (term / 12)
total_amount = principal + total_interest
monthly_payment = total_amount / term
```

**Per-Installment Breakdown** (for each payment 1 to term):
```python
# Calculate interest for current period based on remaining balance
interest_portion = remaining_balance × (annual_rate / 100) / 12

# Calculate principal portion
principal_portion = monthly_payment - interest_portion

# Update remaining balance
new_balance = remaining_balance - principal_portion

# Handle final payment rounding
if payment_number == term:
    # Ensure final balance is exactly 0.00
    principal_portion += remaining_balance
    new_balance = 0.00
```

**Precision Requirements**:
- Use Python `Decimal` type for all monetary calculations
- Round all monetary values to 2 decimal places
- Ensure final payment results in exactly $0.00 balance

### FR4: Response Formatting

- All monetary values formatted to 2 decimal places
- Payment numbers sequential (1, 2, 3, ..., term)
- Installments array length must equal term
- Total payment = sum of all payments in installments array
- Total interest = sum of all interest portions in installments array

## Constraints

### Technical Constraints
- **Simple Interest Only**: Must use simple interest formula, NOT compound interest or standard amortizing loan formula
- **No Database**: All calculations performed in-memory, no data persistence
- **Synchronous Calculation**: Calculation logic is synchronous (no async needed for computation)
- **Development Mode**: CORS must allow localhost origins for frontend integration

### Functional Constraints
- **Fixed Payment Amounts**: All payments (except potentially the last) are equal
- **Monthly Payments Only**: No support for bi-weekly, weekly, or custom payment schedules
- **No Fees**: Calculations exclude origination fees, processing fees, or other charges
- **No Extra Payments**: No support for extra principal payments or early payoff scenarios

### Performance Constraints
- API must handle calculations for terms up to 360 months (30 years)
- Response time should be < 100ms for typical requests
- No rate limiting required (single-user development environment)
