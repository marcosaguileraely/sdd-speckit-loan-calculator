## Feature Description

The Payment Display component is responsible for presenting the calculated monthly payment amount and loan summary metrics to the user after a successful loan calculation. This component receives data from the API response and displays it in a clear, prominent, user-friendly format with proper currency formatting. It serves as the primary result visualization that helps users immediately understand their monthly obligation and total loan costs.

### Context
- Renders after successful API response from loan calculation
- Acts as the primary "answer" to the user's loan inquiry
- Must be visually prominent and easy to understand
- Displays summary metrics to provide comprehensive loan understanding
- Part of the main application flow (Input → Calculate → Display Results)
- On Desktop and Tablet: displayed in the main content area to the right of the LoanForm side-panel
- On Mobile: displayed below the LoanForm in a single-column stacked layout

### Business Value
- Provides immediate, clear answer to user's primary question: "What will my monthly payment be?"
- Helps users understand total loan cost beyond just monthly payment
- Educational value through breakdown of total payment vs. principal vs. interest
- Builds user confidence through clear, professional presentation

## User Stories

### Story 1: View Monthly Payment
**As a** consumer  
**I want to** see my calculated monthly payment amount displayed prominently  
**So that** I immediately understand my monthly financial obligation

**Acceptance Criteria**:
- Monthly payment is displayed in large, readable font
- Amount is formatted as currency (e.g., $856.07)
- Clearly labeled as "Monthly Payment" or similar
- Visible immediately after calculation completes

---

### Story 2: Understand Total Loan Cost
**As a** consumer  
**I want to** see the total amount I'll pay over the life of the loan  
**So that** I understand the full cost including interest

**Acceptance Criteria**:
- Total payment amount is displayed
- Total interest amount is shown separately
- Original principal amount is referenced
- All amounts formatted as currency

---

### Story 3: Review Calculation Summary
**As a** consumer  
**I want to** see a summary of all key loan metrics in one place  
**So that** I can quickly understand the complete picture of my loan

**Acceptance Criteria**:
- Summary includes: monthly payment, total payment, total interest, principal
- Information is organized in logical groups
- Visual hierarchy makes monthly payment most prominent
- All metrics display with consistent currency formatting

## Functional Requirements

### FR-PD-1: Monthly Payment Display
**Requirement**: Display the calculated monthly payment amount prominently

**Specifications**:
- **Data Source**: `monthlyPayment` from API response
- **Format**: US currency format (`$X,XXX.XX`)
- **Styling** (per style guide):
  - **Card Container**: White background (`#FFFFFF`), rounded corners (16px), shadow (`0 2px 8px rgba(0,0,0,0.08)`), padding (32px)
  - **Dollar Sign Icon**: Light gray icon in top-right corner of card
  - **Label**: "Monthly Payment" — 14-15px, Medium weight (500-600), dark gray (`#374151`)
  - **Amount**: 42-48px, Bold (700), near-black (`#1A1A1A`)
  - **Description text**: 14px, Regular (400), gray (`#6B7280`) — below amount
  - **Font Family**: Inter, SF Pro Display, or system sans-serif
- **Precision**: Always show 2 decimal places

**Example Display**:
```
        [$]   ← dollar sign icon (top-right)
Monthly Payment
$856.07
Fixed monthly installment
```

### FR-PD-3: Conditional Rendering
**Requirement**: Component responds to calculation state appropriately

**Specifications**:
- **Before Calculation**: Show placeholder or hide component
- **During Calculation**: Optional loading state
- **After Successful Calculation**: Display all metrics
- **After Error**: Hide component or show appropriate message

**States**:
| State | Component Behavior |
|-------|-------------------|
| No data | Hidden or placeholder text ("Calculate a loan to see results") |
| Data present | Fully rendered with all metrics |
| New calculation | Update values smoothly (no flicker) |

### FR-PD-4: Currency Formatting
**Requirement**: All monetary values must be formatted consistently

**Specifications**:
- **Format**: US locale currency
- **Pattern**: `$X,XXX.XX`
- **Decimal Places**: Always 2
- **Thousands Separator**: Comma (`,`)
- **Currency Symbol**: Dollar sign (`$`)
- **Negative Values**: Should not occur (validation prevents)

**Implementation**: Use JavaScript `Intl.NumberFormat` or similar:
```javascript
const formatter = new Intl.NumberFormat('en-US', {
  style: 'currency',
  currency: 'USD',
  minimumFractionDigits: 2,
  maximumFractionDigits: 2
});
```

### FR-PD-5: Visual Design (per Style Guide)
**Requirement**: Component must follow the project style guide for a modern financial SaaS appearance

**Specifications**:
- **Design Style**: Modern financial SaaS application — clean, minimal interface with generous white space
- **Visual Hierarchy**:
  - Total Payment card (featured): Purple gradient, most visually striking
  - Monthly Payment card: White card with large bold amount, primary "answer" for user
  - Payment Breakdown card: White card, secondary detail information
  - Labels: Clear but subtle, gray (`#6B7280`)
  
- **Spacing**: 24px gaps between cards (per style guide)
- **Internal Padding**: 24-32px within cards
- **Alignment**: Right-align monetary values, left-align labels
- **Shadows**: Subtle shadows for depth (`0 2px 8px rgba(0,0,0,0.08)`) on white cards
- **Border Radius**: 16px consistently across all cards
- **Color Palette**:
  - Featured card: Purple gradient (`#5B4FFF` → `#4A3FE8`)
  - Standard cards: White (`#FFFFFF`)
  - Headings: Near-black (`#1A1A1A`)
  - Body/labels: Medium gray (`#6B7280`)
  - App background: Light gray (`#F5F5F7`)
- **Icons**: Simple line icons, 2px stroke width, positioned in card top-right
- **Responsiveness**: Three-level responsive design across Desktop, Tablet, and Mobile

---

### FR-PD-6: Responsive Layout
**Requirement**: Component adapts to three responsive breakpoints

**Specifications**:

#### Desktop (`≥1024px`)
- Displayed in the main content area (right of the LoanForm side-panel)
- Summary metrics may use a horizontal grid layout (e.g., 3 metrics in a row)
- Monthly payment displayed prominently with ample whitespace
- Full width of the main content area

#### Tablet (`≥768px and <1024px`)
- Displayed in the main content area (right of the narrower LoanForm side-panel)
- Summary metrics may use a 2-column or 3-column grid depending on available width
- Font sizes may be slightly reduced to fit within constrained width
- Maintains visual hierarchy with monthly payment as most prominent

#### Mobile (`<768px`)
- Displayed below the LoanForm in a single-column stacked layout
- Summary metrics stack vertically (one metric per row)
- Monthly payment remains prominent but scaled for mobile readability
- Full viewport width with appropriate padding

## Implementation Notes

### Component Structure
```javascript
// PaymentDisplay.jsx
export function PaymentDisplay({ calculationData }) {
  if (!calculationData) {
    return <div>Calculate a loan to see your payment details</div>;
  }

  const { monthlyPayment, totalPayment, totalInterest } = calculationData;
  
  // Format currency helper
  const formatCurrency = (amount) => {
    return new Intl.NumberFormat('en-US', {
      style: 'currency',
      currency: 'USD'
    }).format(amount);
  };

  return (
    <div className="payment-display">
      <div className="monthly-payment">
        <h2>Monthly Payment</h2>
        <p className="amount">{formatCurrency(monthlyPayment)}</p>
      </div>
      
      <div className="loan-summary">
        <h3>Loan Summary</h3>
        {/* Summary metrics */}
      </div>
    </div>
  );
}
```
