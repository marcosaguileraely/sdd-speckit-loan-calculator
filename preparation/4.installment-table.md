## Feature Description

The Installment Table component is a React component that displays the complete amortization schedule for the calculated loan. It presents a detailed breakdown of each monthly payment, showing how much goes toward principal, how much toward interest, and the remaining balance after each payment. This table is the core educational component of the application, helping users visualize the entire loan repayment journey from first payment to final payoff.

The table receives the installments array from the API response and renders it in a clear, readable format with proper currency formatting. It includes a summary/totals row at the bottom to show aggregate values across all payments.

**Context**: This component works in conjunction with the Payment Display component (Feature #3) to provide comprehensive loan calculation results. While Payment Display shows the high-level summary, the Installment Table provides the granular payment-by-payment details.

## User Stories

### Story 1: View Payment Breakdown
**As a** consumer  
**I want to** see a detailed table showing each monthly payment breakdown  
**So that** I can understand how my loan balance decreases over time

**Acceptance Criteria**:
- Table displays all payments from first to last
- Each row shows payment number, amount, principal, interest, remaining balance, and a progress bar
- All monetary values are formatted as currency
- Progress bar visually shows the proportion of the loan paid off, increasing with each payment (no percentage text)
- Table is visible immediately after calculation completes

---

### Story 2: Understand Interest vs. Principal Distribution
**As a** consumer  
**I want to** see how much of each payment goes to interest vs. principal  
**So that** I can understand the composition of my monthly payment

**Acceptance Criteria**:
- Principal and interest columns are clearly separated
- Values are accurate based on simple interest calculation
- I can see how the interest portion decreases over time
- I can see how the principal portion increases over time

---

### Story 3: Verify Loan Payoff
**As a** consumer  
**I want to** see that the final payment results in a $0.00 balance  
**So that** I can confirm the loan will be fully paid off

**Acceptance Criteria**:
- Final row shows remaining balance of $0.00
- Total row shows sum of all payments equals total payment amount
- Total row shows sum of principal equals original loan amount
- Total row shows sum of interest equals total interest

## Functional Requirements

### FR-4.1: Table Structure
**Priority**: High

The installment table must be implemented as a React component that:
- Renders an HTML `<table>` element with semantic markup
- Includes `<thead>`, `<tbody>`, and `<tfoot>` sections
- Uses proper table accessibility attributes (scope, headers)
- Accepts `installments` prop (array from API response)
- Conditionally renders based on data availability (only show after calculation)

---

### FR-4.2: Table Columns
**Priority**: High

The table must include exactly 6 columns in this order:

| Column # | Header | Data Field | Format | Description |
|----------|--------|------------|--------|-------------|
| 1 | Payment # | `paymentNumber` | Integer | Sequential payment number (1, 2, 3, ..., n) |
| 2 | Payment Amount | `paymentAmount` | Currency | Fixed monthly payment amount |
| 3 | Principal | `principal` | Currency | Portion applied to principal |
| 4 | Interest | `interest` | Currency | Portion applied to interest |
| 5 | Remaining Balance | `remainingBalance` | Currency | Outstanding principal after payment |
| 6 | Progress | (computed) | Progress Bar | Visual-only progress bar showing proportion of loan paid off (no percentage text) |

**Column Specifications**:
- **Payment #**: Right-aligned, no decimals, no currency symbol
- **Payment Amount**: Right-aligned, currency formatted ($X,XXX.XX)
- **Principal**: Right-aligned, currency formatted ($X,XXX.XX)
- **Interest**: Right-aligned, currency formatted ($X,XXX.XX)
- **Remaining Balance**: Right-aligned, currency formatted ($X,XXX.XX)
- **Progress**: Center-aligned, visual progress bar (no percentage text)

**Progress Column Details**:
- **Calculation**: `progressPercent = ((originalLoanAmount - remainingBalance) / originalLoanAmount) * 100`
- **Visual Representation**: HTML progress bar element (`<progress>` or styled `<div>`) showing the percentage filled
- **Label**: No percentage text displayed; the bar is visual-only
- **Color Guidance**: Bar fill uses primary purple (`#5B4FFF`) per style guide; background is light gray (`#E0E0E0`)
- **First Payment**: Shows a small percentage filled (proportional to first principal payment)
- **Final Payment**: Shows 100.0% with fully filled bar
- **Width**: Minimum 100px for the progress bar element to ensure readability
- **Data Dependency**: Requires the original loan amount (passed as a separate prop or derived from the first installment's data)

---

### FR-4.3: Currency Formatting
**Priority**: High

All monetary values must be formatted as currency:
- **Format**: US Dollar format with 2 decimal places
- **Pattern**: `$X,XXX.XX` (e.g., `$856.07`, `$1,234.56`)
- **Zero values**: Display as `$0.00`
- **Negative values**: Should never occur (validation required)
- **Implementation**: Use `Intl.NumberFormat` or equivalent

**Example Formatting**:
```javascript
const formatter = new Intl.NumberFormat('en-US', {
  style: 'currency',
  currency: 'USD',
  minimumFractionDigits: 2,
  maximumFractionDigits: 2
});
```

---

### FR-4.4: Totals/Summary Row
**Priority**: High

The table footer (`<tfoot>`) must include a summary row showing:

| Column | Label/Value | Calculation |
|--------|-------------|-------------|
| Payment # | "Total" | Label only |
| Payment Amount | Sum of all payments | `Σ paymentAmount` |
| Principal | Sum of all principal | `Σ principal` (should equal original loan amount) |
| Interest | Sum of all interest | `Σ interest` (should equal total interest) |
| Remaining Balance | Empty or "—" | No calculation (final balance is $0.00) |
| Progress | Full bar | Fully filled progress bar (loan fully paid); no percentage text |

**Styling**: 
- Summary row should be visually distinct (bold text, top border, background color)
- Clearly separated from payment rows

### FR-4.5: Responsive Design
**Priority**: Medium

The table must be responsive and usable across three breakpoint levels:

**Desktop (`≥1024px`)**:
- Full table with all 6 columns visible, including the Progress bar column
- Displayed in the main content area to the right of the LoanForm side-panel
- Standard table layout with comfortable column widths
- Progress bar renders at full width (minimum 100px)
- No horizontal scroll needed; columns fit within available width

**Tablet (`≥768px and <1024px`)**:
- Full table with all 6 columns visible
- Displayed in the main content area to the right of the narrower LoanForm side-panel
- Progress bar may render at a slightly reduced width but must remain readable
- Columns may use slightly compressed widths to fit the reduced main content area
- Horizontal scroll may activate if content exceeds available width
- Column headers remain visible (sticky header recommended)

**Mobile (`<768px`)**:
- Table occupies full viewport width (no side-panel)
- Horizontal scroll with sticky first column (Payment #) for context
- All 6 columns remain accessible via horizontal scrolling, including Progress
- Progress bar maintains minimum width of 80px for readability on mobile
- Alternative: Card-based layout where each payment renders as a card (optional enhancement)
- Column headers remain visible (sticky header recommended)

**Preferred Approach**: 
- Desktop/Tablet: Standard table layout within the main content area beside the side-panel
- Mobile: Horizontal scroll with sticky Payment # column and sticky header row

### FR-4.6: Data Rendering
**Priority**: High

**Data Source**: `installments` array from API response  
**Data Structure**: Each installment object contains:
```json
{
  "paymentNumber": 1,
  "paymentAmount": 856.07,
  "principal": 814.40,
  "interest": 41.67,
  "remainingBalance": 9185.60
}
```

**Rendering Logic**:
1. Determine the original loan amount (from a prop or from first installment: `installments[0].remainingBalance + installments[0].principal`)
2. Map over `installments` array
3. Create one table row per installment
4. Format monetary values using currency formatter
5. Ensure sequential payment numbers (1 to n)
6. For each row, compute the progress percentage: `((originalLoanAmount - remainingBalance) / originalLoanAmount) * 100`
7. Render a visual-only progress bar in the Progress column (no percentage text displayed)
8. Validate final `remainingBalance` is $0.00 and final progress bar is fully filled

**Edge Cases**:
- **Empty array**: Show message "No payment schedule available"
- **Missing data**: Show error message
- **Final balance not zero**: Display warning (data integrity issue)

**Priority**: Medium

- Progress bar percentage value must be conveyed via `aria-valuenow` for screen readers (no visible text label)

## Constraints

### Technical Constraints
- **Display Only**: Component is read-only (no user interaction beyond viewing)
- **No Persistence**: Data is not saved or stored
- **Single Calculation**: Shows one loan calculation at a time (no comparison)
- **Client-Side Rendering**: Table rendered in browser (no server-side rendering)

### Functional Constraints
- **Fixed Columns**: The table has 6 fixed columns; cannot add/remove columns from table
- **No Sorting**: Payments must be shown in sequential order (cannot sort)
- **No Filtering**: All payments must be displayed (cannot filter by payment number)
- **No Export**: No CSV/PDF export functionality (out of scope for PoC)
- **Simple Interest Only**: Displays results from simple interest calculations only

### Design Constraints
- **Responsive**: Must work across three levels — Desktop (≥1024px), Tablet (≥768px), Mobile (<768px)
- **Side-panel aware**: On Desktop and Tablet, table width adjusts to the main content area beside the LoanForm side-panel
- **Accessibility**: Must meet basic accessibility standards (WCAG AA)
- **Browser Compatibility**: Modern browsers only (Chrome, Firefox, Safari, Edge latest versions)

## Implementation Notes

### Component File Structure
```
src/components/InstallmentTable/
├── InstallmentTable.jsx       # Main component
├── InstallmentTable.test.jsx  # Vitest tests
└── InstallmentTable.css       # Component styles
```

### Props Interface
```javascript
interface InstallmentTableProps {
  installments: Array<{
    paymentNumber: number;
    paymentAmount: number;
    principal: number;
    interest: number;
    remainingBalance: number;
  }>;
  loanAmount: number; // Original loan principal, used to compute progress percentage
}
```

**Progress Calculation Helper**:
```javascript
/**
 * Computes the loan payoff progress percentage.
 * @param {number} loanAmount - Original loan principal
 * @param {number} remainingBalance - Outstanding balance after payment
 * @returns {number} Progress percentage (0 to 100)
 */
const calcProgress = (loanAmount, remainingBalance) =>
  ((loanAmount - remainingBalance) / loanAmount) * 100;
```
