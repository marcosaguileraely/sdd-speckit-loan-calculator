## Feature Description

This specification describes three focused visual adjustments to the existing application layout:

1. **Card Reorder**: Swap the horizontal positions of the **Total Payment** card (currently left) and the **Monthly Payment** card (currently right) within the Payment Display component, so that **Monthly Payment appears on the right** and **Total Payment appears on the left**.
2. **Vertical Purple Toolbar**: Add a narrow vertical toolbar on the far-left edge of the application shell, using the primary purple color (`#5B4FFF`), containing a single icon that navigates to (or highlights) the Loan Simulator feature.
3. **Toast Notifications**: Add a toast notification system to display transient user messages — green for informational/success messages (e.g., "Calculation completed successfully") and red for error messages (e.g., "Failed to calculate: invalid interest rate").

These changes amend the layout described in `preparation/3.payment-display.md` and formalize the vertical navigation bar already referenced in `preparation/styling.md` ("VERTICAL NAVIGATION: Narrow sidebar on far left (~76px) with purple background").

**As a** consumer  
**I want to** see the Total Payment card on the left and the Monthly Payment card on the right  
**So that** the most comprehensive cost metric leads visually and monthly payment is the natural "answer" on the right

**Acceptance Criteria**:
- Total Payment (purple gradient card) renders in the **left column** of the two-card top row
- Monthly Payment (white card) renders in the **right column** of the two-card top row
- Payment Breakdown card remains below, spanning the full width of the two-card row
- Card visual styles are unchanged (only position swaps)

---

### Story SA-2: Vertical Purple Toolbar
**As a** consumer  
**I want to** see a vertical purple toolbar on the far left of the screen with a Loan Simulator icon  
**So that** I have a persistent, branded navigation anchor point for the application

**Acceptance Criteria**:
- A narrow vertical toolbar (`76px` wide) with a solid purple background (`#5B4FFF`) is always visible on the far-left edge of the viewport
- A white calculator/loan-simulator icon is centered horizontally and positioned near the top of the toolbar
- Clicking or tapping the icon activates/highlights the Loan Simulator (scrolls to or focuses the loan input form)
- The toolbar does not collapse or hide at any breakpoint (it may reduce in width on mobile — see responsive section)

---

### Story SA-3: Toast Notifications
**As a** consumer  
**I want to** see a brief notification when a calculation completes or an error occurs  
**So that** I have clear, immediate feedback about the result of my actions without navigating away from my current view

**Acceptance Criteria**:
- A green toast appears at the top-center of the viewport when a calculation succeeds, displaying a success message
- A red toast appears at the top-center of the viewport when an error occurs, displaying the error message
- Each toast auto-dismisses after a short duration (4s info, 6s error)
- Toasts can be manually dismissed by clicking the `×` button
- Toasts do not block or overlay interactive elements (the user can continue using the form)

### FR-SA-2: Vertical Purple Toolbar

**Requirement**: Render a fixed, narrow vertical toolbar on the far-left edge of the application viewport.

**Specifications**:

#### Dimensions & Position
- **Width**: `76px`
- **Height**: `100vh` (full viewport height)
- **Position**: Fixed, `left: 0`, `top: 0`, `z-index: 100` (above page content)
- **The main application content area** must be offset by `76px` from the left to prevent overlap

#### Visual Design
- **Background**: Solid purple `#5B4FFF`
- **No border** on the right side — the purple itself acts as the separator
- **No shadow** on this toolbar (flat purple surface)
- **Border Radius**: None (flush to viewport edges)

#### Icon — Loan Simulator
- **Icon**: Calculator / loan simulator icon (white, filled or 2px stroke line style)
  - Recommended: a simple calculator outline icon from any standard icon library (e.g., Heroicons `calculator`, Lucide `calculator`, or equivalent)
  - Size: `28px × 28px`
  - Color: `#FFFFFF`
- **Position within toolbar**: Horizontally centered (`margin: 0 auto`), vertically positioned `24px` from the top of the toolbar
- **Active/hover state**: Icon background circle `rgba(255,255,255,0.15)`, border-radius `50%`, padding `10px` — providing subtle visual feedback
- **Tooltip**: On hover (desktop), show a tooltip reading `"Loan Simulator"` — dark background (`#1A1A1A`), white text, `12px`, positioned to the right of the icon

#### Interaction Behavior
- **Click / Tap**: Scrolls the page to the Loan Input Form (smooth scroll to `#loan-form` or equivalent anchor), OR if the form is already visible, sets focus on the first input field
- **Active state**: When the Loan Simulator section is in view, the icon background highlights (`rgba(255,255,255,0.20)`)
- **Keyboard**: Icon is focusable (`tabIndex={0}`) and activates on `Enter` or `Space`
- **ARIA**: `role="navigation"`, `aria-label="Application navigation"` on the toolbar; icon button has `aria-label="Go to Loan Simulator"`

#### Content
- Only one icon is present in this version — the Loan Simulator icon
- No text labels inside the toolbar at any breakpoint
- No logo or brand mark inside the toolbar

### FR-SA-3: Toast Notification System

**Requirement**: Provide a toast notification component to display transient user-facing messages for events such as calculation completion, validation errors, and API failures.

**Specifications**:

#### Visual Design

| Property | Info Toast (Success) | Error Toast |
|----------|---------------------|-------------|
| Background | `#16A34A` (green-600) | `#DC2626` (red-600) |
| Text Color | `#FFFFFF` | `#FFFFFF` |
| Icon | Checkmark circle (`✓`) | Exclamation circle (`!`) |
| Icon Color | `#FFFFFF` | `#FFFFFF` |
| Border Radius | `8px` | `8px` |
| Box Shadow | `0 4px 12px rgba(0,0,0,0.15)` | `0 4px 12px rgba(0,0,0,0.15)` |
| Min Width | `320px` | `320px` |
| Max Width | `480px` | `480px` |
| Padding | `14px 20px` | `14px 20px` |

#### Layout & Position
- **Position**: Fixed, `top: 24px`, horizontally centered in the viewport (`left: 50%; transform: translateX(-50%)`)
- **z-index**: `1000` (above all other content including the toolbar)
- **Content layout**: Horizontal row — icon (left), message text (center/fill), dismiss button (right)
- **Typography**: `14px`, Medium (500), `#FFFFFF`
- **Dismiss button**: `×` character, `#FFFFFF`, `20px`, transparent background, `border: none`, cursor pointer; `aria-label="Dismiss notification"`

#### Animation
- **Enter**: Slide down from top + fade in, `300ms ease-out`
- **Exit**: Slide up + fade out, `200ms ease-in`
- CSS transition on `opacity` and `transform: translateY()`

#### Auto-Dismiss
- **Info toasts**: Auto-dismiss after `4000ms` (4 seconds)
- **Error toasts**: Auto-dismiss after `6000ms` (6 seconds) — longer duration so users can read error details
- **Manual dismiss**: User can click the `×` button to dismiss immediately at any time
- **Timer pause**: Auto-dismiss timer pauses on hover (desktop) and resumes on mouse leave

#### Stacking
- If multiple toasts are triggered in quick succession, they stack vertically with a `12px` gap between them
- Maximum `3` toasts visible simultaneously; older toasts are dismissed when the limit is exceeded
- Each toast manages its own auto-dismiss timer independently

#### Toast Types & Usage

| Type | Background | Use Case Examples |
|------|-----------|-------------------|
| `info` | `#16A34A` (green) | "Calculation completed successfully", "Loan details updated" |
| `error` | `#DC2626` (red) | "Failed to calculate: invalid interest rate", "Network error — please try again", "Loan amount must be greater than zero" |

#### Component API (React)

```tsx
// Toast types
type ToastType = 'info' | 'error';

interface Toast {
  id: string;
  type: ToastType;
  message: string;
}

// Hook API for triggering toasts
interface UseToast {
  showToast: (type: ToastType, message: string) => void;
  dismissToast: (id: string) => void;
}
```

- Toasts are managed via a `useToast` custom hook (or React context provider at the App shell level)
- `showToast('info', 'Calculation completed successfully')` — shows a green info toast
- `showToast('error', 'Loan amount must be greater than zero')` — shows a red error toast

#### Accessibility
- Toast container has `role="status"` and `aria-live="polite"` for info toasts
- Toast container has `role="alert"` and `aria-live="assertive"` for error toasts
- Dismiss button is keyboard accessible (`tabIndex={0}`, activates on `Enter` or `Space`)
- Toasts do not trap focus and do not interfere with the current focus context
- Color contrast ratio meets WCAG AA: white text on green/red backgrounds exceeds 4.5:1

#### Responsive Behavior
- **Desktop / Tablet**: Toast centered at top, `min-width: 320px`, `max-width: 480px`
- **Mobile (`<768px`)**: Toast stretches to full viewport width minus `32px` total horizontal margin (`margin: 0 16px`), centered at top

### FR-SA-5: Responsive Behavior

**Requirement**: Both the reordered cards, the toolbar, and toast notifications adapt across the three breakpoints.

#### Desktop (`≥1024px`)
- Two-card top row: `grid-template-columns: 1fr 1fr` — Total Payment left, Monthly Payment right
- Payment Breakdown card: full-width below (`grid-column: 1 / -1`)
- Toolbar: `76px` wide, full height, visible

#### Tablet (`≥768px and <1024px`)

  3. Payment Breakdown card (third)

### Toast Component (React component sketch)

```tsx
// ToastContext.tsx
import { createContext, useContext, useState, useCallback, useRef } from 'react';

type ToastType = 'info' | 'error';

interface Toast {
  id: string;
  type: ToastType;
  message: string;
}

interface ToastContextValue {
  showToast: (type: ToastType, message: string) => void;
  dismissToast: (id: string) => void;
}

const ToastContext = createContext<ToastContextValue | null>(null);

export function useToast(): ToastContextValue {
  const ctx = useContext(ToastContext);
  if (!ctx) throw new Error('useToast must be used within a ToastProvider');
  return ctx;
}

export function ToastProvider({ children }: { children: React.ReactNode }) {
  const [toasts, setToasts] = useState<Toast[]>([]);
  const timersRef = useRef<Map<string, ReturnType<typeof setTimeout>>>(new Map());

  const dismissToast = useCallback((id: string) => {
    setToasts((prev) => prev.filter((t) => t.id !== id));
    const timer = timersRef.current.get(id);
    if (timer) { clearTimeout(timer); timersRef.current.delete(id); }
  }, []);

  const showToast = useCallback((type: ToastType, message: string) => {
    const id = crypto.randomUUID();
    const duration = type === 'error' ? 6000 : 4000;

    setToasts((prev) => {
      const next = [...prev, { id, type, message }];
      return next.length > 3 ? next.slice(next.length - 3) : next;
    });

    const timer = setTimeout(() => dismissToast(id), duration);
    timersRef.current.set(id, timer);
  }, [dismissToast]);

  return (
    <ToastContext.Provider value={{ showToast, dismissToast }}>
      {children}
      <div className="toast-container">
        {toasts.map((toast) => (
          <div
            key={toast.id}
            className={`toast toast-${toast.type}`}
            role={toast.type === 'error' ? 'alert' : 'status'}
            aria-live={toast.type === 'error' ? 'assertive' : 'polite'}
          >
            <span className="toast-icon">
              {toast.type === 'info' ? '✓' : '!'}
            </span>
            <span className="toast-message">{toast.message}</span>
            <button
              className="toast-dismiss"
              aria-label="Dismiss notification"
              onClick={() => dismissToast(toast.id)}
            >
              ×
            </button>
          </div>
        ))}
      </div>
    </ToastContext.Provider>
  );
}
```
